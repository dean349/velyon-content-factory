Portfolio Scanner & Curator — User Guide

Full reference for the Portfolio Scanner section of the Velyon Website Content Factory Studio. Covers Discover → Curate → Generate, the dual-catalog architecture, cross-linking, and every tab in the editor.

1. The Big Picture: Three Modes

The Portfolio Scanner has three modes, selected via tabs at the top of the panel:

🔍
Discover
Scan Vercel, GitHub, URLs for real projects
→
✏️
Curate
Fill in the story — metrics, team, tech stack
→
⚡
Generate
Batch-produce case studies & content exports

The pipeline is sequential. An item goes: scanned (empty shell) → curated (you fill it in) → generation-ready (you flip its visibility and it appears in Generate mode).

2. Two Catalogs, Not One

The Portfolio Scanner manages two separate catalogs in a single interface:

📋
Case Studies
Work delivered for a client — the proof Velyon can execute. Examples: "AI Fraud Detection Pipeline for [Bank]", "RAG Search System for [Legal Firm]"
🚀
Velyon Products
Proprietary AI tools/systems Velyon built and owns — your own IP, not client work. Examples: Velyon Validator Loop, any internal tool.

They live side by side in the Curate list, but they are structurally distinct: case-study-specific tabs (Client Intake, Transformation, Delivery, etc.) only appear when editing a Case Study, and the Product Profile tab only appears when editing a Velyon Product.

How to switch between them

In the editor header (top of the page), you'll see two toggle buttons:

  • 📋 Case Study — indigo when active
  • 🚀 Velyon Product — emerald when active

Click to switch. The available tabs change automatically — tabs that don't apply simply disappear. If you were on a tab that's no longer visible, you'll drop back to Overview automatically.

3. What Just Happened When You Clicked a Project

Vercel's API gave the scanner exactly four things about that project: its name, its live URL, its framework (e.g. React/Vite), and a last-deployed timestamp. Everything else is empty — no metrics, no client name, no testimonials, no team, no screenshots. That's normal.

New items default to Case Study. If you know it's a Velyon product (not client work), switch the toggle to 🚀 Velyon Product immediately before filling anything in.

4. Recommended Fill-In Order

The tabs render left-to-right in a fixed order, but that's not the best order to fill them in.

For a Case Study

  1. Overview — name it, tag it, describe it
  2. Client Intake — capture the "why" before you forget it
  3. Tech Stack — verify/correct what was auto-detected
  4. Metrics — your hero numbers
  5. Transformation — before/after evidence (optional)
  6. Testimonials — quotes, if you have any (optional)
  7. Assets — screenshots, diagrams
  8. Team — who worked on it
  9. Delivery / Methodology / Approval / Performance / Competitive / Containers / Post-Launch — fill in whichever apply
  10. Content Hints — tell the AI generator what angle to take
  11. Redaction — decide what's public vs. internal
  12. Template / Exports — configure output format
  13. Related Products — pick which Velyon products powered this case study
  14. Comments — flag anything for follow-up
  15. Save

For a Velyon Product

  1. Overview — name it, describe what it does
  2. Product Profile — delivery model, maturity stage, use cases, capabilities
  3. Tech Stack — verify/correct what was auto-detected
  4. Metrics — internal performance numbers, if applicable
  5. Assets — screenshots, diagrams (optional)
  6. Team — who built it
  7. Methodology / Performance / Containers — if applicable
  8. Content Hints — what angle to take for the website
  9. Redaction — visibility level
  10. Export Configs — output format (if ready)
  11. Related Cases — auto-populated, read-only
  12. Comments — flag anything, for follow-up
  13. Save

5. Tab-by-Tab Reference

Core Tabs Both Catalogs

📄 Overview

The front page of the entry.

  • Tagline — one-line value prop
  • Status — production / beta / alpha / deprecated / archived / internal
  • Description — short summary
  • Long Description — the full narrative (multi-paragraph)
  • Tech Stack Summary — quick tag view of frontend/backend/AI/infra/monitoring (full editing in Tech Stack tab)
  • Industry / Capability / Product Tags — freeform tags with autocomplete suggestions
  • Methodology Phases — freeform tag field
  • Related Products / Related Cases — see Section 6 (Cross-Linking)

🏆 Metrics

Your hero numbers — the stats that sell the entry.

  • Each metric has: Label, Value, Context, Category (speed/cost/accuracy/adoption/revenue/efficiency/latency/throughput/quality/custom)
  • Estimation Method — say so honestly if a number isn't measured
  • Confidence Interval (Min/Max) — for estimated numbers
  • ★ Hero toggle — mark your single best metric as the headline stat
  • ✓ Verified toggle — mark whether confirmed or provisional
  • Click the comment icon on any metric to flag it as missing/unverified

🖼️ Assets

Screenshots, diagrams, videos.

  • Type: screenshot / architecture-diagram / video-demo / logo / ui-component / data-viz / testimonial-video / code-snippet / wireframe / flowchart / dashboard-capture
  • URL, Alt Text, Caption
  • Redacted Version URL — a blurred/anonymized variant for when visibility = Redacted
  • ★ Hero toggle for the lead image

👥 Team

Who built/delivered it.

  • Name, Role, Contribution, Avatar URL, LinkedIn, Email
  • Velyon Team checkbox — distinguishes your people from client-side stakeholders
  • Hours Invested, Period start date

🧱 Tech Stack

Ten categories: frontend, backend, ai, infrastructure, monitoring, database, messaging, auth, testing, cicd.

  • Each entry has a Confidence level: 🔍 Detected / 🧠 Inferred / ✋ Manual / ✨ Fabricated
  • Source — where the entry came from (package.json, header, Dockerfile, user-input, ai-generated, etc.)
  • The Vercel scan auto-populates frontend, infrastructure, monitoring, and cicd — everything else is manual

🔒 Redaction

Controls how the entry appears publicly. Four levels:

LevelClient NameLogoMetricsAssets
Public✅✅✅✅

If you pick Redacted, you also set: Archetype (e.g. "Tier-1 Investment Bank"), Approved Assets, Hide/Anonymize Metrics.

🎯 Content Hints

Steers the AI when it generates content.

  • Suggested Format — technical-deep-dive / business-outcome / transformation-story / product-showcase / architecture-review
  • Target Audiences — cto, vp-eng, ml-engineer, product-manager, investor, peer
  • Key Technical Insights, Replicable Patterns, Notable Constraints, SEO Keywords, Competitive Differentiators — all tag fields

💬 Comments

An annotation trail on the entry itself.

  • Comment types: missing-info, auto-generate, fabricated, needs-review, legal-hold, technical-debt, client-feedback, general
  • Use this to leave notes like "need real conversion numbers before this goes public" — resolve them once addressed

🚀 Product Profile Tab Velyon Products Only

This tab only appears when catalogType = "Velyon Product". It captures what makes Velyon's own IP distinct:

  • Delivery Model (multi-select):
    • Consulting-Led — Velyon team implements it directly
    • Co-Build — built alongside the client's own engineers
    • License — client licenses it and runs it themselves
    • Managed — Velyon hosts and operates it on the client's behalf
  • Maturity Stage: concept → alpha → beta → production → sunset
  • Target Use Cases — tag field (e.g. "Fraud detection", "Real-time inference")
  • Key Capabilities — tag field (e.g. "Sub-100ms latency", "Self-healing pipelines")
  • Internal Only checkbox — when checked, this product won't appear on the public velyon.io /products page
Note: Pricing tiers are intentionally not included yet — this is deferred.

6. Cross-Linking: Case Studies ↔ Velyon Products

This feature connects your two catalogs so that:

  • A case study can declare "this was built using Product X" (Overview tab → Related Products multi-select)
  • A product automatically shows "this product was used in Case Study A and B" (Overview tab → Related Cases, read-only)

Case Study side (editable)

  • In the Overview tab, find "🚀 Velyon Products Used In This Case Study"
  • Click + Add to search through all Velyon Products
  • Pick one or more — each becomes a tag; click x to remove
  • When you Save, the system automatically builds the reverse link

Product side (auto-derived, read-only)

  • In the Overview tab, find "📋 Case Studies Built With This Product"
  • This fills itself in automatically — you don't (and can't) edit it manually
  • It updates every time you save any case study that declares that product

Why this is one-directional by design

You only ever declare the link from case study → product. The reverse direction (product → case studies) is a computed derivation: the system walks through every case study, sees which products it references, and writes the reverse link. This prevents the two sides from getting out of sync and means you only have to do the work once.

7. Critical Gotcha: Visibility Defaults to "Internal Only"

Every freshly-scanned item defaults to internal-only visibility.
Generate mode only shows items where visibility !== 'internal-only'.

If you curate a project fully but forget to change its visibility on the Redaction tab, it will never appear in Generate mode — it'll look like it silently disappeared. This applies equally to both Case Studies and Velyon Products.

Fix: Go to Redaction → pick Public, Client Approved, or Redacted before switching to Generate mode.

For Velyon Products specifically: the Product Profile tab also has an "Internal Only" checkbox — this is a separate gate controlling whether the product appears on the public velyon.io /products page.

8. Saving

Click the Save button (top-right of the editor, next to the visibility dropdown). This:

  • Writes an entry to the item's enrichmentHistory (audit trail)
  • Re-syncs the bidirectional cross-links between case studies and products
  • Closes the editor and returns you to the project list
  • Everything is stored in your browser's localStorage under key velyon_portfolio_items — not synced to a server, so don't clear your browser data without exporting first

If the editor crashes, you'll see a red error box with a Try Again button instead of a blank screen.

9. Generate Mode — Batch Content Creation

Once items have a non-internal visibility:

  1. Switch to ⚡ Generate
  2. Check the boxes next to every project you want to generate content for
  3. Click Generate N Case Studies

This calls the synthesis engine, which builds a content prompt per item from everything you filled in — tech stack, metrics, methodology, team, content hints, redaction level — and queues it for the content factory pipeline.

10. Adding Items Manually (Without Scanning)

Not all Velyon products or case studies will come through the scanner. Use the manual add buttons in the Curate list:

  • + Case Study — creates a new case study and opens it in the editor
  • + Velyon Product — creates a new product with a blank Product Profile and opens it

You'll be prompted for a name, then dropped straight into the editor.
